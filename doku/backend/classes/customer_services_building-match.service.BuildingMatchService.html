    Class BuildingMatchService

    Service für das Matching von Kundenadressen gegen die normalisierte Gebäudereferenz (tp_gis.gebref_norm).

    Remarks

    • Nutzt ausschließlich serverseitige Normalisierung (unaccent, tg_norm_, tg_house_no_), damit Frontend/Backend einheitlich bleiben.
    • Aktuell wird nur die exakte T0-Variante abgedeckt (schnell und präzise).
    • Falls kein Treffer gefunden wird, wird null zurückgegeben (kein Error-Throw).

      ): Promise<null | GebRefMatch>

      Versucht, eine Adresse gegen die Gebäudereferenz zu matchen (T0: perfekter Treffer).

      Parameters

      • str: null | string

        Straßenname (unsaniert; wird in SQL normalisiert)

      • hnr: null | string

        Hausnummer (kann Zahl+Suffix enthalten, z. B. "14a")

      • adz: null | string

        Adresszusatz (optional; z. B. "Hinterhaus", "a", …)

      • ort: null | string

        Ort/Kreis (wird mit kreis_norm verglichen; Benennung historisch bedingt)

      • Optionalplz: null | string

        (optional) Postleitzahl – Hinweis: kommt in gebref_norm nicht vor (derzeit ungenutzt)

      Returns Promise<null | GebRefMatch>

      Ein GebRefMatch bei Erfolg, sonst null.

      Example

      const hit = await buildingMatchService.match('Musterstraße', '12a', null, 'Musterstadt');
      if (hit) {
        console.log(hit.oid, hit.lon, hit.lat);
      }
